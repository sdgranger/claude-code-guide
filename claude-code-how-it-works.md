# Claude Code 작동 방식 가이드

> 출처: [Claude Code의 작동 방식](https://code.claude.com/docs/ko/how-claude-code-works)
> 에이전트 루프, 내장 도구, 컨텍스트 관리를 이해합니다.

---

## 1. 에이전트 루프 (Agent Loop)

Claude Code는 단순한 챗봇이 아닌 **에이전트**입니다.
작업을 주면 세 가지 단계를 반복합니다:

```
[프롬프트] → 컨텍스트 수집 → 작업 수행 → 결과 검증 → (반복) → [완료]
                ↑                                    ↓
                └──────── 사용자 개입 가능 ────────────┘
```

### 핵심 개념

| 단계 | 하는 일 | 예시 |
|------|---------|------|
| **컨텍스트 수집** | 코드를 읽고, 파일을 검색하고, 구조 이해 | `grep`, `read`, 파일 탐색 |
| **작업 수행** | 파일 편집, 명령 실행, 코드 생성 | `edit`, `bash`, `write` |
| **결과 검증** | 테스트 실행, 빌드 확인, 오류 수정 | `npm test`, `./gradlew build` |

> 루프는 작업에 맞게 조정됩니다.
> - 질문 → 컨텍스트 수집만
> - 버그 수정 → 세 단계 반복
> - 리팩토링 → 광범위한 검증 포함

- [ ] 에이전트 루프의 3단계 (수집 → 수행 → 검증) 이해하기
- [ ] `Esc` 키로 중간에 개입하여 방향 수정해보기

## 2. 모델 (Models)

Claude Code는 여러 모델을 사용할 수 있습니다.

| 모델 | 모델 ID 예시 | 특징 | 사용 시나리오 |
|------|-------------|------|-------------|
| **Sonnet** | `claude-sonnet-4-6` | 빠르고 균형잡힌 성능 | 대부분의 코딩 작업 (기본 추천) |
| **Opus** | `claude-opus-4-7` | 강력한 추론, 복잡한 문제 해결 | 아키텍처 결정, 어려운 버그, plan |
| **Haiku** | `claude-haiku-4-5-20251001` | 빠르고 경제적 | 간단한 작업, subagent, 코스트 최적화 |

```bash
# 세션 중 모델 전환
/model

# 시작 시 모델 지정
claude --model opus
```

- [ ] `/model`로 모델 전환해보기
- [ ] `Option+P` / `Alt+P`로 빠른 모델 전환 해보기

## 3. 내장 도구 (Built-in Tools)

도구가 Claude를 **에이전트**로 만듭니다. 도구 없이는 텍스트만 출력할 수 있습니다.

| 범주 | Claude가 할 수 있는 것 | 예시 |
|------|----------------------|------|
| **파일 작업** | 읽기, 편집, 생성, 이름 변경 | 코드 수정, 새 파일 생성 |
| **검색** | 패턴 매칭, 정규식 검색 | 파일 찾기, 코드 검색 |
| **실행** | 셸 명령, 서버, 테스트, git | `npm test`, `git commit` |
| **웹** | 검색, 문서 가져오기 | 오류 조회, API 문서 확인 |
| **코드 인텔리전스** | 타입 오류 확인, 정의 이동, 참조 찾기 | 플러그인 필요 |

### 도구 사용 예시: "실패한 테스트를 수정해"

```
1. 테스트 실행 → 실패 확인              (실행 도구)
2. 오류 출력 읽기                       (파일 도구)
3. 관련 소스 파일 검색                   (검색 도구)
4. 파일 읽어서 코드 이해                 (파일 도구)
5. 파일 편집하여 수정                    (파일 도구)
6. 테스트 다시 실행하여 검증              (실행 도구)
```

> 각 도구 사용은 다음 단계를 결정하는 새로운 정보를 반환합니다.

- [ ] Claude에게 "테스트 실행해줘" 요청하여 도구 사용 과정 관찰하기
- [ ] `Ctrl+O`로 Thinking 보기 활성화하여 Claude의 추론 과정 확인

## 4. Claude가 접근할 수 있는 것

`claude`를 실행하면 다음에 접근할 수 있습니다:

| 접근 대상 | 설명 |
|-----------|------|
| **프로젝트** | 현재 디렉토리 및 하위 디렉토리의 파일 |
| **터미널** | 빌드 도구, git, 패키지 관리자, 스크립트 등 |
| **git 상태** | 현재 브랜치, 커밋되지 않은 변경 사항, 최근 커밋 |
| **CLAUDE.md** | 프로젝트별 지침, 규칙, 컨텍스트 |
| **자동 메모리** | Claude가 자동으로 저장한 학습 내용 |
| **확장 기능** | MCP 서버, Skills, Subagents 등 |

> **인라인 코드 어시스턴트와의 차이점**: Claude Code는 현재 파일만이 아닌 **전체 프로젝트**에서 작업할 수 있습니다.

- [ ] `@파일명`으로 특정 파일을 참조하여 질문해보기
- [ ] Claude에게 "이 프로젝트의 구조를 설명해줘" 요청해보기

## 5. 세션 관리

### 세션의 독립성
각 새 세션은 이전 대화 기록 없이 **새로운 컨텍스트 윈도우**로 시작합니다.
세션 간 지식 전달은 CLAUDE.md와 자동 메모리를 통해 이루어집니다.

### 세션 재개 & 포크

| 명령 | 동작 |
|------|------|
| `claude --continue` | 가장 최근 대화 재개 |
| `claude --resume` | 대화 선택기에서 선택 |
| `claude --continue --fork-session` | 대화를 복제하여 새 세션 시작 |

```
원본 세션: [메시지1] → [메시지2] → [메시지3]
                                      ↓
포크 세션:  [메시지1] → [메시지2] → [메시지3] → [새로운 방향]
```

- [ ] `claude --continue`로 이전 대화 이어보기
- [ ] `/rename 세션명`으로 세션에 이름 지정해보기
- [ ] `claude --resume`으로 세션 선택기 사용해보기

## 6. 컨텍스트 윈도우 (가장 중요한 리소스!)

컨텍스트 윈도우 = 대화 기록 + 파일 내용 + 명령 출력 + CLAUDE.md + Skills + 시스템 지침

### 컨텍스트가 가득 차면?

```
1단계: 오래된 도구 출력 지우기
2단계: 대화 요약 (자동 압축)
→ 요청과 주요 코드는 유지, 대화 초반 세부 지침 손실 가능
```

### 컨텍스트 관리 전략

| 도구/명령 | 용도 |
|-----------|------|
| `/clear` | 관련 없는 작업 전환 시 완전 초기화 |
| `/compact <지시>` | 압축 시 유지할 내용 지정 |
| `/context` | 컨텍스트 사용 현황 확인 |
| `/btw` | 사이드 질문 (컨텍스트에 추가되지 않음) |
| **Subagent** | 별도 컨텍스트에서 작업 → 요약만 반환 |
| **CLAUDE.md** | 지속적 규칙은 여기에 (압축에도 유지됨) |

> **CLAUDE.md는 압축을 견딥니다!** `/compact` 후 Claude가 디스크에서 CLAUDE.md를 다시 읽습니다.
> 대화에서만 전달한 지침은 압축 시 손실될 수 있으므로 중요한 규칙은 반드시 CLAUDE.md에 작성하세요.

- [ ] `/context`로 컨텍스트 사용 현황 확인해보기
- [ ] `/compact`로 압축 실행해보기
- [ ] `/btw`로 사이드 질문 (컨텍스트 절약) 해보기

## 7. 체크포인트 & 권한 (안전 메커니즘)

### 체크포인트
- Claude가 파일 편집 **전에** 현재 내용의 스냅샷을 만듦
- `Esc` × 2 또는 `/rewind`로 이전 상태 복원 가능
- 파일 변경만 다룸 (외부 시스템 작업은 체크포인트 불가)

### 권한 모드

| 모드 | 동작 | 전환/시작 |
|------|------|----------|
| **default** | 모든 도구 사용 전 승인 요청 | `Shift+Tab` |
| **acceptEdits** | 파일 편집 + 일반 fs 명령(`mkdir`/`mv`/`cp` 등) 자동, 그 외 요청 | `Shift+Tab` |
| **plan** | 읽기 전용 (분석/계획만, 변경 없음) | `Shift+Tab` 또는 `/plan` |
| **auto** | 백그라운드 안전 분류기로 모든 작업 자동 진행 | 옵트인 후 `Shift+Tab` 또는 `--permission-mode auto` |
| **dontAsk** | 사전 승인된 도구만 허용, 그 외는 자동 거부 (CI용) | `--permission-mode dontAsk` |
| **bypassPermissions** | 보호 경로 외 모든 검사 생략 (격리 환경용!) | `--dangerously-skip-permissions` |

```
기본 사이클: default → acceptEdits → plan → (활성화된 모드들) → default
```

> **`auto` 모드**는 Sonnet 4.6/Opus 4.6/Opus 4.7만 지원 (Haiku 미지원). 분류기가 3회 연속/누적 20회 차단하면 default로 폴백.
> **어떤 모드든** [protected paths](https://code.claude.com/docs/en/permission-modes#protected-paths) 쓰기는 자동 승인되지 않음 (`.claude/`, `~/.claude/`, repo의 git 메타 등).

### 권한 허용 목록 (`.claude/settings.json`)
```json
{
  "permissions": {
    "allow": [
      "Bash(npm test)",
      "Bash(git *)",
      "Read"
    ]
  }
}
```

- [ ] `Shift+Tab`으로 권한 모드 순환해보기
- [ ] `Esc+Esc`로 체크포인트 되돌리기 해보기
- [ ] `.claude/settings.json`에 자주 쓰는 명령 허용 목록 추가해보기

## 8. 효과적인 사용 팁

### 대화형으로 사용하기
```
시작: "로그인 버그 수정"
[Claude가 조사]
수정: "세션 처리 쪽 문제야"
[Claude가 방향 조정]
```
> 완벽한 프롬프트가 필요 없습니다. 시작하고 반복하세요.

### 처음부터 구체적으로
```
나쁜 예: "빌드 실패해"
좋은 예: "체크아웃 흐름이 만료된 카드 사용자에게 손상됨.
         src/payments/ 토큰 리프레시 확인. 실패 테스트 먼저 작성 후 수정해."
```

### 지시하지 말고 위임하기
```
나쁜 예: "user.ts 파일을 열고, 35번 줄의 함수를 찾아서, null 체크를 추가해"
좋은 예: "사용자 조회에서 NPE가 발생해. 조사하고 수정해줘"
```
> 읽을 파일이나 실행할 명령을 지정할 필요 없습니다. Claude가 파악합니다.

- [ ] 모호한 프롬프트 대신 구체적인 프롬프트 작성 연습하기
- [ ] "조사하고 수정해줘" 패턴으로 위임해보기
- [ ] Claude에게 검증 방법 제공하기 (테스트, 스크린샷, 예상 출력)
